Aula 7: Otimização e regressão não-linear¶

Objetivos¶

  • Ajustar curvas que no podem ser transformadas em regresses lineares.

  • Explorar mtodos de otimizao

Otimização¶

  • Otimizar: melhorar a eficincia, reduzir custos, etc.
  • Envolve:
    • Definir uma funo objetivo, a ser minimizada (ou maximizada);
    • Incorporar restries a essa funo objetivo;
    • Mtodos para encontrar o(s) mnimo(s) (ou mximo(s)) da funo objetivo.
  • Exemplos:
    • Encontrar o retngulo de maior rea com um certo permetro dado.
    • Determinar o menor caminho passando por determinados pontos uma nica vez (caixeiro viajante).
    • Maximir a produo de energia (eltrica, solar, elica, ...).
    • Minimizar perdas em vrios processos.
    • Mnimos quadrados:
      • Funo objetivo a ser minimizada a soma dos quadrado dos resduos.
      • Ou pode ser vista como maximizao da verossimilhana.

Otimização com o python¶

O pacote scipy.optimize (Optimization and Root Finding) possui vrias funes de otimizao:

  • minimize_scalar: minimizao de funes escalares de uma varivel.
  • minimize: minimizao (local) com ou sem restries de funes objetivo escalares de uma ou mais variveis.
  • least_squares: mnimos quadrados no-linear, com ou sem restries.
  • curve_fit: ajuste de curvas baseado na funo de mnimos quadrados no-linear acima.
  • nnls e lsq_linear: mnimos quadrados linear com restries.
  • linprog: minimizao de funes objetivo lineares, com restries lineares, para programao linear.
  • basinhopping, brute, differential_evolution, shgo, dual_annealing: algoritmos variados de minimizao global.

Observao: O numpy.linalg.lstsq que temos usado uma implementao direta do mtodo de mnimos quadrados sem restrio.

Ajuste de curva via mínimos quadrados não-linear¶

A funo objetivo de mnimos quadrados no linear uma verso no linear dos mnimos quadrados.

Mais precisamente, dada uma funo de ajuste $f_\beta$, com $\beta=(\beta_1, \ldots, \beta_k)$ representando os parmetros dessa funo, e dado uma amostra $(x_i, y_i)_i$, $i=1, \ldots, N$, temos os resduos $\Delta_i= y_i - f_\beta(x_i)$ e o objetivo

$$ \text{minimizar } F(\beta) = \frac{1}{2} \sum_i \rho\left(\left(y_i - f_\beta(x_i)\right)\right)^2, \; \text{com uma restrio} \; l_\beta \leq \beta \leq u_\beta. $$

Opções para a função de "perda"¶

H vrias funes $\rho$ implementadas e possvel usar funes definidas pelo usurio:

  • "linear" (default), com $\rho(z) = z$: mnimos quadrados clssico).

  • "soft_l1", com $\rho(z) = 2 ((1 + z)^{1/2} - 1)$. Aproximao suave para a norma $L^1$, visando diminuir a influncia de outliers.

  • "huber", com $\rho(z) = z$, se $z <= 1$, ou $2z^{1/2} - 1$. Similar ao "soft_l1".

  • "cauchy", com $\rho(z) = \ln(1 + z)$. Diminui severamente a influncia de outliers, mas dificulta o processo de otimizao.

  • "arctan" : $\rho(z) = \arctan(z)$. Similar ao "cauchy".

Mimetizando as funções de perda¶

In [9]:
z = np.linspace(0, 4, 200)
z_sq = z**2
rho = dict()
rho.update({"linear": z_sq})
rho.update({"soft_l1": 2*((1+z_sq)**0.5 - 1)})
rho.update({"huber": np.minimum(z, 2*((1+z_sq)**0.5-1))})
rho.update({"cauchy": np.log(1+z_sq)})
rho.update({"arctan": np.arctan(z_sq)})
rho.update({"l1": z})

Funções de ajuste no scipy.optimize¶

Temos, de incio, trs opes para implementar a minimizao da soma do quadrado dos resduos $\Delta_i = y_i - f_\beta(x_i)$,

$$ \text{minimizar } F(\beta) = \frac{1}{2} \sum_i \rho\left(\left(y_i - f_\beta(x_i)\right)\right)^2, \; \text{com uma restrio} \; l_\beta \leq \beta \leq u_\beta. $$

  • Usar a funo minimize, que mais flexvel, mas devemos escrever explicitamente a funo objetivo $F(\beta)$, que pode ser da forma acima ou diferente.

  • Usar a funo least_squares, onde passamos explicitamente a escolha de $\rho$ e um vetor com os resduos $\Delta_i$, que podem ser $y_i - f_\beta(x_i)$ ou diferente.

  • Usar a funo curve_fit, onde definimos $\rho$ e passamos separadamente a funo de ajuste $f_\beta$ e os dados $x_i$, $y_i$, para que a funo curve_fit monte a funo objetivo $F(\beta)$ e chame a funo least_squares para fazer o ajuste.

Métodos de otimização das funções¶

Os mtodos de otimizao da funo minimize foram mencionados acima.

Os mtodos das funes least_squares e curve_fit so

  • "trf" : Trust Region Reflective algorithm, particularly suitable for large sparse problems with bounds. Generally robust method. Default if bounds are given.

  • "dogbox" : dogleg algorithm with rectangular trust regions, typical use case is small problems with bounds. Not recommended for problems with rank-deficient Jacobian.

  • "lm" : Levenberg-Marquardt algorithm as implemented in MINPACK. Doesnt handle bounds and sparse Jacobians. Usually the most efficient method for small unconstrained problems. Default if bounds are not given.

# lstsq do numpy
A_np = np.vstack([x,np.ones(num_points)]).T
m_np, c_np = np.linalg.lstsq(A_np, y, rcond=None)[0]

# curve_fit do scipy
def f(x, m, c):
    return m*x+c
m_cf, c_cf = sp.optimize.curve_fit(f, x, y)[0]

# least_squares do scipy
def residuo(a, x, y):
   return f(x,a[0],a[1]) - y
m_ls, c_ls = sp.optimize.least_squares(residuo, [1.1,0.0], args=(x,y)).x

# minimize do scipy

def objetivo(a, x, y):
    return np.linalg.norm(residuo(a,x,y))**2
m_mn, c_mn = sp.optimize.minimize(objetivo, [1.1,0.0], args=(x,y)).x

Função de perda e outliers¶

Nas funes do scipy, o default uma funo de perda linear.

Na presena de outliers, podemos tentar identificar e remover esses outliers ou diminuir a influncia deles.

A funo de perda implementa essa segunda opo.

Vamos incluir dois outliers no exemplo anterior e variar a funo de perda.
